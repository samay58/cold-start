LESSON 0002 · COLD START

How Cold Start Builds Source Bedrock

The card is only as good as the sources allowed into extraction. This lesson follows the actual code path from domain to accepted evidence.

The one sentence version

Cold Start does not ask the model to research from memory. It gathers public and provider evidence, rejects sources that do not fit the target company, ranks what remains, then asks the LLM to extract only facts that can point back to citations.

Main path: apps/web/src/app/api/generate/route.ts -> apps/web/src/inngest/functions.ts -> apps/web/src/inngest/source-fetching.ts -> packages/pipeline/src/source-gate.ts -> packages/pipeline/src/evidence-ledger.ts -> packages/llm/src/extraction.ts

The full path

1. Normalize the target

The API turns the input into a canonical company domain and slug. Initial extension generation starts as a basics profile run; analysis and section jobs happen later.

2. Build a research plan

The current worker uses `fallbackResearchPlan(domain)`. That creates fixed queries for funding, profile, team, signals, comparables, and independent analysis.

3. Fetch cheap direct evidence

Direct Exa searches for company profile, people, funding, and recent news. These are fast fundamentals.

4. Fetch paid fallback evidence

StableEnrich adds Exa search, similar-company search, Firecrawl homepage/about/team pages, and org enrichment. Basics uses the faster subset; later analysis or gated section work can use the fuller path.

5. Skip duplicate paid probes

If direct Exa already covers an intent, cheap-first mode can skip overlapping StableEnrich Exa probes. This protects budget.

6. Merge sources by URL

Direct and StableEnrich results are deduped by URL. The system wants a single source bed, not several copies of the same article.

7. Gate sources by target fit

The source gate accepts or rejects every source. Bad protocol, wrong company site, same-name wrong domain, and low-relevance sources are cut out before extraction.

8. Abort if the bedrock is empty

If no source survives the gate, the run returns an error. The pipeline does not spend LLM tokens on empty evidence.

9. Build an evidence ledger

Accepted sources are grouped by canonical URL, assigned authority scores, given snippets, and sorted by usefulness.

10. Extract cited facts

The LLM receives the ledger plus bounded raw source text. It must return structured fields, citations, and nulls for missing evidence.

11. Patch weak blocks

If description, funding, team, signals, or comparables are thin, late enrichment can fetch more sources and run a narrower block extraction.

12. Sanitize trust

Facts without valid citations are nulled. Synthesis without visible supported citation markers is stripped. Public output removes synthesis and person emails.

What counts as a source

Shape Meaning Where defined
ProviderSource A fetched item with URL, title, source type, fetched time, raw text, and optional retrieval intent. packages/providers/src/types.ts
ProviderFactCandidate A structured fact from a provider, such as HQ, founder, headcount, or funding, with a citation URL and confidence. packages/providers/src/types.ts
Citation The card-level source object that facts point to by ID. packages/core/src/card.ts
Provider facts can help fill missing fields, but they do not bypass trust. They become citations with IDs like p1, and downstream trust rules still apply.

How sources get accepted or rejected

The source gate lives in packages/pipeline/src/source-gate.ts. It is intentionally blunt.

Rule Accepted Rejected
Protocol HTTP and HTTPS URLs. Enrichment records can pass even without a normal URL. Unsupported protocols for normal web sources.
Company site match A company-site source on the target domain or its subdomain. A company-site source from another domain.
Wrong same-name domain Trusted hosts like TechCrunch, BusinessWire, LinkedIn, GitHub, and major VC sites. A suspicious domain with the same or nearly same root label as the target.
Target mention Sources whose title or raw text mention the target domain or root company label. Sources that neither match the target host nor mention the target.
Comparables Comparable sources get a special pass because competitor pages may not mention the target. They still need later extraction to prove real overlap.

How source quality is understood

Cold Start separates relevance from quality. A source can be relevant but weak. Source quality lives in packages/core/src/source-quality.ts.

Tier Best use Risk
Independent technical Product depth, technical differentiation, real constraints. May be narrow or opinionated.
Independent analysis Market structure and investor framing. May be thesis-shaped.
Independent report Funding, dates, announcements, external confirmation. Can repeat company positioning.
Primary company Product mechanics, official pages, exact company claims. Weak for evaluative claims.
Press release Exact announcement facts. Promotional by design.
Enrichment Coverage and missing firmographics. Lower confidence unless corroborated.

What the LLM is allowed to do

The LLM does not get an open-ended research task. It gets accepted sources, an evidence ledger, and a tool schema.

  • It must map material facts to citation IDs.
  • It must use null when evidence is missing.
  • It must build funding from round evidence before using totals.
  • It must emit one signal per event, not one signal per article.
  • It must treat source incentives as part of the evidence.
  • It must surface conflicts as mixed instead of averaging.
The important idea: sources do not become truth just because they were fetched. They become candidate evidence. Citation checks, source quality, schema validation, and trust sanitization decide what survives.

Review prompt
