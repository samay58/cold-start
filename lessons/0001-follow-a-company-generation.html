LESSON 0001 · COLD START

Follow one company generation from request to stored card

By the end of this lesson, you should be able to explain the main write path in plain English and know which file to open when something in generation behaves oddly.

The product idea behind the code

Cold Start exists to answer one question quickly: is this company worth the next ten minutes of investor attention?

That goal creates the architecture. The system needs a fast public card first, because a useful sourced profile is valuable even before deep analysis. Then the extension can ask for stronger investor synthesis once enough evidence exists.

So the code is split into two lanes: basics for the sourced public profile, and analysis or section:{id} for gated investor reads.

The main path

1. RequestPOST /api/generate checks the domain, mode, auth, confirmation, cache, and active runs.
2. QueueThe route writes a queued generation row, records an event, and sends an Inngest event.
3. WorkgenerateCardFunction fetches sources, calls the pipeline, records traces, and saves the result.
4. Buildgenerate-card.ts extracts facts, merges provider facts, enriches weak blocks, and validates the card.
5. Storerepository.ts writes the card JSON, evidence, sources, TTLs, sections, and run state.
6. ReadThe public API strips private synthesis. The extension API returns the full card only after auth.

Walk the code

  1. Start at apps/web/src/app/api/generate/route.ts. The key idea is policy: should this request be allowed, served from cache, blocked, or queued?
  2. Move to apps/web/src/inngest/functions.ts. The key idea is orchestration: one worker owns the run, records events, calls providers, and injects LLM functions into the pipeline.
  3. Open packages/pipeline/src/generate-card.ts. The key idea is correctness: build a typed card, gate synthesis, and drop unsupported claims.
  4. Open packages/core/src/card.ts. The key idea is contract: a non-null fact without citation IDs is invalid.
  5. Open packages/db/src/repository.ts. The key idea is persistence: cards.card_json is the source of truth, with freshness and run state around it.

The most important mental model

Think of generation as a factory with inspectors at every station.

The provider layer brings in raw material. The LLM extracts structured facts. The pipeline checks whether the facts have citations. The trust layer strips or downgrades weak claims. The public route removes private synthesis. The extension can show more only when authenticated.

The product promise is not "we always fill every field." The promise is: if the product says something, it can show where it came from.

Answer key

Start with apps/web/src/app/api/generate/route.ts if the question is about whether a run was queued, cached, blocked, or already active. Move to apps/web/src/inngest/functions.ts if the run exists but the worker behavior is unclear. Move to apps/extension/src/research-layer.ts only when the backend data looks right but the panel is presenting it oddly.
