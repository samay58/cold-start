Cold Start priority brief | May 9, 2026

Stabilize the trust loop before adding more surface area.

The product idea is strong and the repo is no longer a blank prototype. The next pass should prove the card is accurate on real companies, make generation failures recoverable, and turn the side panel into the real product surface.

Recommendation

Tackle three things first: real-card evaluation, generation reliability, and side-panel cohesion. Slug policy and production hardening come right behind them because they become painful once public cards exist.

What I re-grounded on

Product truth

Basics first

The spec is clear: identity, domain, team, funding, signals, comparables, and citations matter before synthesis. Judgment only earns space after the facts hold.

Trust model

Citations are the product

Public pages must never expose synthesis. Unsupported facts should become empty or unknown, not confident filler.

Session pattern

Stumbles were operational

The hard failures were stale extension origin, missing auth config, Inngest worker gaps, stale Next build cache, and stuck generation rows.

Priority stack

01

Run a real-card evaluation sprint.

This is the highest leverage work because the product dies if fundamentals are wrong. The 50-company seed exists. Use it to generate, score, and classify misses instead of debating architecture in the abstract.

eval/golden-companies.seed.json | eval/README.md | packages/pipeline/src/generate-card.ts

Done when

We have a scored table for at least the first 10 to 15 companies: identity, funding, leadership, citation validity, public synthesis leak check, extension synthesis check, latency, and provider failure reason.

02

Make generation failures visible and recoverable.

We already hit the real failure mode: a stale queued or running row can block retry and force manual SQL. Fixing this will save future testing sessions from turning into archaeology.

apps/web/src/app/api/generate/route.ts | packages/db/src/repository.ts | docs/deployment.md

Done when

A run status/debug surface shows queued, running, failed, provider failures, timestamps, and cost. Stale runs can be retired safely without hand-editing the database.

03

Resolve slug policy before public corpus growth.

Current slugs use the first hostname label, so two domains can collide. This is a public URL product. It is cheaper to fix before many cards, screenshots, and links exist.

packages/core/src/slug.ts | packages/db/src/schema.ts | apps/web/src/app/c/[slug]/page.tsx

Done when

`foo.com` and `foo.ai` cannot collide. Existing pretty slugs either remain intentionally supported or redirect cleanly from a documented legacy path.

04

Reconcile the design source of truth, then polish the side panel.

Session history says Master Plate Monograph became the code direction. Current docs still carry older observatory-shell language in places. Pick the source of truth, update it, then do the thorough side-panel pass.

DESIGN.md | docs/brand/semitechie-vc-design-ethos.md | packages/ui/src/tokens.css | apps/extension/src/styles.css

Done when

The extension side panel, generating state, source drawer, public card, and OG image read as one product. Screenshots pass at extension width, mobile web, and desktop web using real cards.

05

Make synthesis failure an honest product state.

The verifier is correctly conservative. The UI should not look broken when too few claims survive. Show a clean "not enough verified evidence" state and record why claims dropped.

packages/llm/src/verifier.ts | packages/pipeline/src/generate-card.ts | packages/ui/src/SynthesisSection.tsx

Done when

Analysis either shows supported lines with citations or explains that evidence was insufficient. Unsupported drops are inspectable in debug records, not lost in logs.

06

Harden production only after the above is moving.

Env sync, Vercel, Neon, extension ID, dependency audit, and token handling matter. They should follow the product proof pass, not replace it.

SECURITY.md | docs/deployment.md | apps/web/src/lib/extension-auth.ts

Done when

Production env validation is explicit, audit upgrades have a tested path, and the deployed extension cannot accidentally use local sentinel auth.

Open items that matter

Item Why it matters Decision or action
Golden eval coverage The spec names fundamentals coverage as the first reason Cold Start wins. Run the seed list, score failures, and use the failure table to drive provider or extraction fixes.
Stale generation rows We already had to clear stuck rows manually during dev testing. Add stale-run retirement and a local/debug run view before more live testing.
Slug collision The public card URL is the artifact. Collisions create trust and sharing problems. Move now to a domain-disambiguated policy or consciously accept pretty-slug migration work later.
Design drift Current code and docs do not describe the same visual system cleanly. Make Master Plate Monograph or the observatory-shell system the explicit current source of truth, then polish.
Cost observability The pipeline has cost fields, but live provider and LLM costs may stay zero without wiring. Record cost lines during runs and show them in the debug view.
Dependency audit Known advisories exist, but blind force fixes would be risky. Track as deliberate upgrade work after the product proof loop starts.

What not to do next

Do not build watchlists, public generation, web-side synthesis auth, X bot behavior, contact enrichment, or new launch pages yet. The repo already has enough ambition. The next win is a narrow card that is fast, cited, recoverable, and visually credible.
