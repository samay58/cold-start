# Provider Cost Assumptions

This file is the source trail for hardcoded provider cost estimates. These numbers are for run telemetry and budget guards, not invoice-grade accounting.

| Surface | Code location | Current assumption | Source |
|---|---|---|---|
| Direct Exa Search | `packages/providers/src/direct-exa.ts` | `$0.007` per successful Search request. Exa lists Search at `$7 / 1k requests`, so `7 / 1,000 = 0.007`. The current Direct Exa requests stay at or under 10 results with text and highlights. | `https://exa.ai/pricing`, `https://exa.ai/docs/changelog` |
| Exa Websets people email | `packages/providers/src/websets.ts` | `15` credits per accepted person item: 10 credits for a matching result plus 5 credits for email or phone enrichment. Default credit price is `$49 / 8,000 = $0.006125`. | `https://websets.exa.ai/websets/billing` |
| StableEnrich via AgentCash | `packages/providers/src/provider-budget.ts` | `$0.01` for search/scrape-style probes and `$0.02` for most org or people enrichment probes. These are budget estimates, not official provider list prices. | Reconcile with AgentCash wallet deltas and endpoint traces before using for pricing decisions. |
| StableEnrich analysis re-fetch policy (`ANALYSIS_SOURCE_REFRESH`) | `apps/web/src/inngest/source-fetching.ts` (`analysisSourceFetchPlan`, `fetchInitialSourcesForGeneration`), `apps/web/src/inngest/worker-env.ts` (`analysisSourceRefreshModeFromProcess`) | Governs the analysis-mode fetch-sources step only on the `reuseExistingForAnalysis` branch (extraction being reused from an existing investor-usable card). `full` (default; nothing changes until this is explicitly promoted) keeps today's unconditional 13-probe stableenrich fetch. `targeted` narrows every reuse run to the 3-probe signals group (`exa_recent_signals`, `exa_customer_proof`, `exa_independent_analysis`), saving roughly `$0.08-0.12` in `costUsdAgentcash` and ~20s of wall time per run. `skip-fresh` (promoted to production 2026-07-22 on 15 shadow pairs: claims net zero, citations down 5.2% with no gate impact, AgentCash spend fully eliminated) skips the stableenrich fetch entirely and substitutes stored sources (`findSourcesBySlug` via `providerSourcesFromStoredSources`) when the card's signals TTL (6h) is still fresh; measured savings are roughly `$0.13` and ~10s of wall per reuse run (the probe fetch overlaps Direct Exa, so only the serial tail is saved, not the earlier ~30s estimate); it falls back to the targeted group when signals have gone stale. Direct Exa (`providers.directExa.estimatedCostUsd`), Anthropic (`costUsdAnthropic`), and Websets (`providers.websets.estimatedCostUsd`) spend are unaffected in every branch: only the AgentCash-billed stableenrich probes are gated. Trace field `providers.stableenrich.analysisSourceRefresh` records which branch a run actually took. | `.superpowers/sdd/task-5.3-brief.md`; Task 5.1's finding that `verifySynthesis` only ever reads `card.citations` is why this fetch can be narrowed without touching synthesis or verify. |

Update this file when provider pricing, plan tier, request shape, or budget policy changes.
