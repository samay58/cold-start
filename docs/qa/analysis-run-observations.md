# Analysis Run Observations

A dated log of notable production and shadow analysis runs, and the attack list they feed. One entry per observation, newest first within each section. Add entries whenever run monitoring surfaces something worth acting on; move an item to Resolved when it ships. The two-week latency measurement (started 2026-07-22, review around 2026-08-05, bar p50 60s / p90 90s via `npm run measure:analysis-latency`) reads this file.

## Attack list

Ordered by expected impact.

1. **Queue hold before sources land.** The uniqlo run sat 32s between `generation.started` and `source.found` while the wait surface showed Queue. Part is dispatch (below), part is plan plus early fetch running before the first source event. Two angles: emit an earlier Gather-worthy event (`plan.ready` currently maps to no stage; mapping it to Gather splits the hold), and shrink the real pre-source work. UI mapping change is cheap; measure first whether the hold is common (query `research_run_events` gaps).
2. **Inngest dispatch latency.** Baseline: mean 13.6s, p50 7.5s, outliers 47s and 73s (timescaledb). No app-side lever exists (Inngest v4 serve options expose none; verified 2026-07-22). Attack path: Inngest account-tier conversation, or an edge trigger. Out of code's reach today; keep measuring. Escalated 2026-07-23: an Inngest Cloud incident held two organic basics runs (opendoor 290s, tenex 188s of pure queue-side delay; see run log) with the panel showing a dead Queued the whole time. Two consequences worth pursuing: the first paint depends on a third-party dispatcher with no SLA we hold, and the UI has no honest state for an abnormal queue hold (no `generation.started` after ~30s should say so instead of sitting silent).
3. **skip-fresh promotion: DONE (2026-07-22, promoted).** Samay approved on 15 prod-parity pairs: claims 31 to 31 net zero (4 drops, 6 ties, 5 increases, every drop traced to synthesis non-determinism), citations down 5.2% with zero gate impact, AgentCash 100% eliminated, wall down 10.1s median. `ANALYSIS_SOURCE_REFRESH=skip-fresh` is live in Vercel production; rollback is removing the var. The two-week measurement watches citation counts as the tripwire.
4. **Targeted-mode dedup optimism.** The unpromoted `targeted` refresh mode ignores the Direct Exa coverage dedup, so its projected savings are slightly high. Only matters if `targeted` or the skip-fresh stale fallback ever promotes.
5. **Transient-vs-semantic step-failure classing: partial.** Wave 1 shipped `isTransientLlmError` (`packages/llm/src/transient-error.ts`) and wired it into the synthesize-card and verify-synthesis steps (`generation-helpers.ts`, commit `1a8e44e`): a transient 429/5xx/network failure now re-throws so Inngest retries it instead of permanently memoizing a failed step. Residual closed same day: the `generate-section` step now applies the same split (re-throw transient, memoize semantic), wired by inspection against the tested wave-1 pattern since the inline closure has no cheap unit seam.
6. **sidepanel-ui Playwright project is deliberately manual-only.** `npm run qa:extension:ui` (`playwright test --project=sidepanel-ui`) is not part of `apps/extension`'s `test` script, so it never runs inside `npm run check` or CI; it is a by-hand pre-merge gate. The fly-by intent-gate rework (commit `890fa0d`, stop racing wall-clock time against the budget) makes that manual gate trustworthy again instead of flaky. Keep treating it as manual, not automated.

## Run log

Event ULIDs prove the Event API accepted both instantly; run ULIDs show Inngest created the runs at 07:45:21 and 07:45:27 in one backlog sweep — 290s and 188s of pure dispatch delay, 4x the worst outlier in the item-2 baseline. Inngest's status page carried an active "Function Processing Delays" incident at the time (root cause identified, remediation deployed). Once dispatched, both runs completed clean on the current deploy (tenex ~45s compute), exonerating the overnight commits. Feeds attack list item 2.
- **2026-07-23, issue #10 resolved: preserve-on-withhold shipped alongside the all-claims-dropped convergence.** A gate block or an all-claims-dropped verify result no longer overwrites a card that already carries a filed `synthesis` read; the run completes without storing, leaving the existing read in place. When no prior read exists, an all-claims-dropped run now stores a withheld card with the new `no-claims-survived` reason and completes instead of throwing, so a re-click reads the standing verdict for free rather than re-paying for synthesis and failing the same way (closes the former attack list's all-claims-dropped item). Verifier `max_tokens` raised to 8192 in the same pass (closes the former attack list's verifier truncation item).
- **2026-07-22, wave-1 fix: marketStructureAndTiming null-tolerance shipped.** `synthesisSchema` now coerces a null `marketStructureAndTiming` container to `undefined` before validation (`packages/core/src/card.ts`, commit `d4a3112`). ## Resolved

- Silent synthesis withholds (11 of 51 runs): fixed by Phase 1's floor-plus-advisory gate and visible withheld state, deployed 2026-07-21.
- Per-tick full-card polling waste: event-gated since Phase 4 (`de8d048`).
- The 50s event dead zone during synthesis and verify: real events since the Phase 4 step split.
- Verifier truncation exposure: `max_tokens` raised from 2000 to 8192 in `verifier.ts:116`, 2026-07-23.
- All-claims-dropped runs presenting as expensive, non-converging failures: the run now converges instead of throwing, storing a `no-claims-survived` withheld card when no prior read exists or preserving an existing read otherwise (issue #10), 2026-07-23.
