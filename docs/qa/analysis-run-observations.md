# Analysis Run Observations

A dated log of notable production and shadow analysis runs, and the attack list they feed. One entry per observation, newest first within each section. Add entries whenever run monitoring surfaces something worth acting on; move an item to Resolved when it ships. The two-week latency measurement (started 2026-07-22, review around 2026-08-05, bar p50 60s / p90 90s via `npm run measure:analysis-latency`) reads this file.

## Attack list

Ordered by expected impact.

1. **Queue hold before sources land.** The uniqlo run sat 32s between `generation.started` and `source.found` while the wait surface showed Queue. Part is dispatch (below), part is plan plus early fetch running before the first source event. Two angles: emit an earlier Gather-worthy event (`plan.ready` currently maps to no stage; mapping it to Gather splits the hold), and shrink the real pre-source work. UI mapping change is cheap; measure first whether the hold is common (query `research_run_events` gaps).
2. **Inngest dispatch latency.** Baseline: mean 13.6s, p50 7.5s, outliers 47s and 73s (timescaledb). No app-side lever exists (Inngest v4 serve options expose none; verified 2026-07-22). Attack path: Inngest account-tier conversation, or an edge trigger. Out of code's reach today; keep measuring.
3. **skip-fresh promotion: DONE (2026-07-22, promoted).** Samay approved on 15 prod-parity pairs: claims 31 to 31 net zero (4 drops, 6 ties, 5 increases, every drop traced to synthesis non-determinism), citations down 5.2% with zero gate impact, AgentCash 100% eliminated, wall down 10.1s median. `ANALYSIS_SOURCE_REFRESH=skip-fresh` is live in Vercel production; rollback is removing the var. The two-week measurement watches citation counts as the tripwire.
4. **Verifier truncation exposure (latent).** `verifySynthesis` returns plain JSON text against a hardcoded `max_tokens: 2000` (`packages/llm/src/verifier.ts:116`). Sonnet-routed verify truncates on claim-rich companies (6 shadow-run failures under the local misrouted rig); prod DeepSeek verify has never truncated. Pre-existing, unchanged by the step split. Attack options, Samay sign-off required since verifier parameters sit inside the untouchable line: raise the cap, or move verify to forced tool choice like every other stage. Doc guard added: `docs/anthropic-llm-call-map.md`'s verifier purpose entry now warns that any `LLM_VERIFIER_MODEL` change must confirm response fit before shipping.
5. **All-claims-dropped runs present as failures, expensively.** The all-claims-dropped throw fires before any store, so no `synthesisWithheld` record exists and every user re-click queues a fresh full run that re-pays synthesis (~$0.28) and fails identically. Gate-block is cheap and terminal; all-dropped is expensive and non-converging. Product decision, adjacent to issue #10.
6. **Targeted-mode dedup optimism.** The unpromoted `targeted` refresh mode ignores the Direct Exa coverage dedup, so its projected savings are slightly high. Only matters if `targeted` or the skip-fresh stale fallback ever promotes.
7. **Transient-vs-semantic step-failure classing: partial.** Wave 1 shipped `isTransientLlmError` (`packages/llm/src/transient-error.ts`) and wired it into the synthesize-card and verify-synthesis steps (`generation-helpers.ts`, commit `1a8e44e`): a transient 429/5xx/network failure now re-throws so Inngest retries it instead of permanently memoizing a failed step. Residual closed same day: the `generate-section` step now applies the same split (re-throw transient, memoize semantic), wired by inspection against the tested wave-1 pattern since the inline closure has no cheap unit seam.
8. **sidepanel-ui Playwright project is deliberately manual-only.** `npm run qa:extension:ui` (`playwright test --project=sidepanel-ui`) is not part of `apps/extension`'s `test` script, so it never runs inside `npm run check` or CI; it is a by-hand pre-merge gate. The fly-by intent-gate rework (commit `890fa0d`, stop racing wall-clock time against the budget) makes that manual gate trustworthy again instead of flaky. Keep treating it as manual, not automated.
