# Mission: Learn Cold Start From The Inside

## Why
Samay wants to understand Cold Start as both a product and a codebase, so he can reason about it, change it, debug it, and keep shaping it without treating the repo as a black box.

## Success looks like
- Explain what happens when a company profile is generated, from request to stored card.
- Know where to look first for product behavior, schema changes, provider issues, extension state, and public/private data boundaries.
- Read a trace or failing run and form a grounded hypothesis about what broke.
- Make scoped changes that preserve the trust contract: cited facts, public redaction, gated synthesis, and verifier drops.

## Constraints
- Use plain English first, then code.
- Tie implementation details back to product purpose.
- Prefer the live repo and local docs over abstract architecture talk.
- Teach in small lessons that build toward working fluency.

## Out of scope
- Rebuilding the product from scratch.
- Deep frontend visual design history before the main runtime path is understood.
- External framework tutorials unless a repo change needs them.
