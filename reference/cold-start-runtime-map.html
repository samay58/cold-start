Cold Start Runtime Map

The shortest honest map: Cold Start turns a domain into one cited card, stores that card, shows public facts on the web, and lets the extension request gated investor analysis when the evidence is strong enough.

One Company Request

  1. requestPOST /api/generate validates domain, mode, auth, cache, and active runs.
  2. queueThe route writes a queued generation_runs row and sends card/generate.requested to Inngest.
  3. workergenerateCardFunction marks the run running, records events, fetches sources, and calls the pipeline.
  4. pipelinegenerate-card.ts extracts cited facts, merges provider facts, enriches thin blocks, and validates the card.
  5. storerepository.ts writes cards.card_json, evidence rows, sources, TTLs, and run traces.
  6. readThe public page strips synthesis. The extension can read the full card after auth.

Trust Contract

  • Every non-null material fact needs citations. This is enforced by the Zod schema in packages/core/src/card.ts.
  • Public reads remove synthesis and emails. That happens through publicCard() in packages/core/src/trust.ts.
  • Weak synthesis is dropped. The synthesis gate and verifier live in packages/pipeline/src/generate-card.ts.
  • Stale data is explicit. The repository marks cached cards as stale when TTLs expire.

Public vs Extension

  • /api/cards/{slug} returns public facts only.
  • /api/extension/cards/{slug} requires extension identity and token, then returns the full card.
  • Research modules are partly derived from the card and partly stored as research_sections.
  • openQuestions and theCase render from consolidated synthesis, not separate section jobs.

Where To Look First

  • packages/core/src/card.ts for data shape and citation validation.
  • apps/web/src/app/api/generate/route.ts for request policy and queueing.
  • apps/web/src/inngest/functions.ts for orchestration.
  • packages/pipeline/src/generate-card.ts for generation correctness.
  • packages/db/src/repository.ts for storage, cache, run state, and events.
  • apps/extension/src/research-layer.ts for what the reader sees.

Ask the agent to walk any file from top to bottom when a concept is fuzzy. The fastest way to learn this repo is to keep tying each code branch back to the product promise: useful facts first, cited evidence always, investor synthesis only when gated and supported.
